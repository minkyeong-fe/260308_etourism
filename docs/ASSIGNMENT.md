# 요금 계산 엔진(Pricing Engine) 과제

## 1. 과제 내용

### 과제 문구

**요금 계산 엔진(Pricing Engine) 구현**

- **과제**: 아래 조건에 맞춰 최종 견적가를 리턴하는 함수(Function)를 작성해 소스 코드를 보내주세요. (언어: React/Node.js/Java 중 택 1)
- **조건**: [투숙일, 객실타입, 인원] 입력 시 → [해당 기간 시즌/요일 요금 + 조식 × 인원 + 엑스트라베드]를 계산하여 합계 출력
- **데이터**: 첨부된 엑셀 데이터 참조

### 참조 데이터 (엑셀 기준)

| 평균가격 | 지역 | 룸타입 | 주중 비수기 | 주말 비수기 | 주중 성수기 | 주말 성수기 | 극성수기 | 조식 평균가 | 엑스트라베드 평균가격 |

| 4성 | 경주 | 디럭스 | ₩176,000 | ₩187,000 | ₩174,000 | ₩221,667 | ₩268,750 | ₩23,439 | ₩42,000 |

## 2. 구현 정리

### 입력/출력

- **입력**: 체크인일, 체크아웃일, 객실타입(3종 중 1개), 인원, (선택) 엑스트라베드 대수
- **출력**: 최종 견적 합계 및 (선택) 항목별 breakdown(숙박/조식/엑스트라베드)

함수 시그니처 예:

- `calculateQuote(input: PricingInput, seasonConfig?: SeasonConfig): PricingResult`

### 데이터 구조

- **객실 요금표**: `src/data/pricingTable.ts`에 3행 데이터를 `PricingRow` 타입 배열로 정의. 객실 식별은 `3성_서울_스탠다드` | `4성_경주_디럭스` | `5성_부산_스위트` 세 가지.
- **시즌 설정**: `src/config/seasonConfig.ts`에 비수기/성수기/극성수기 구간과 주말 요일을 설정으로 두고, 기본값 사용 시 인자 생략 가능. 수정 시 설정만 바꿔서 주입.

### 계산 로직

1. 체크인~체크아웃 전일까지 1박씩 날짜 목록 생성.
2. 각 1박에 대해: 날짜로 시즌(비수기/성수기/극성수기)과 요일(주중/주말) 판별 후, 해당하는 요금 컬럼 값을 합산.
3. 조식: `조식평균가 × 인원 × 박수`.
4. 엑스트라베드: `엑스트라베드단가 × 엑스트라베드 대수` (전체 숙박 기준 1회 적용).
5. 합계 = 숙박 합계 + 조식 + 엑스트라베드.

### 가정 사항

- 시즌 구간은 **설정 파일**로 관리(비수기/성수기/극성수기). DB 없음. 기본값: 극성수기 7–8월, 성수기 3–6·9–10월, 비수기 1–2·11–12월.
- 주중 = 월~목, 주말 = 금~일. 극성수기는 요금 컬럼이 하나뿐이므로 해당 1박 요금만 사용.
- 엑스트라베드는 대수를 입력받아 `단가 × 대수`로 반영(기본 0).

### 실행 방법

```bash
npm install
npm run dev   # 개발 서버 (예: http://localhost:5173)
npm run build # 프로덕션 빌드
npm run preview # 빌드 결과 미리보기
```

브라우저에서 폼에 투숙일·객실타입·인원·엑스트라베드 대수를 입력 후 견적 확인.
